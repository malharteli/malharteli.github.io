Website Redesign

So, as you might have noticed, I have here a redesigned website. Now, I'm attaching a few pictures to further discuss the material at hand. First of all, we have the first image, image (a). In the first image, you'll find that I had created a very... basic website. Nothing really fancy about it, so when I went into redesigning it, I decided to start from scrath, so to speak.

In image (b), I opened up Adobe Illustrator and began to meddle with menu icons and the layout. At this stage I have not taken mobile platforms into consideration, despite their growing prevelance. For the color scheme, I decided to use a web called Paletton which provides an easy color picker. I ended up using this sandy layout for my initial palette, just to get a feel for the layout. Replacing the colors, I find, will be the easiest part of this whole ordeal.

The hardest thing to get right was the Navigation bar at the top of the page. As image c showcases, just getting the Nav system to stretch across the screen was an ordeal, much less the balance and placement of the icons on the element. Up till now, my website was built on pre-made code provided in the form of Bootstrap, which was accompanied by a css and coressponding javascript file. Bootstrap is characterized by its emphasis on softer corners, and given my focus upon squares for this web design exercise, I found myself overriding a lot of the classes that I refer to throughout the page. As you can see in Image D, I successfully replaced these buttons, though the division they were placed in remained locked to a page-centered margin, rather than spanning across the screen as intended.

In all honesty, these icons were the second easiest part of improving this sight. With a limited palette, I had to communicate as much as I could in a simple image as possible. Thankfully, once they were made in Illustrator, I could export them as SVGs. Now, SVGs can be incredibly complicated, if you do not read the appropriate literature. They have their own classes and can by manipulated with CSS if pasted into the page. But the file size dramatically decreased, and the ability to modify the colors of the objects with additional images is a god-send. Through in transitions in css, and you can pull off some incredible effects, like the color changes when you have over the icons. The borders change color and it transitions quite fluidly thanks to css3's animations.

Now, the buttons, I feel, communicate a great deal with just their appearance. But this does not guarantee a fluid experience with the site. Thus, I needed to account for usability, and include appropriate name tages, as seen in Image b. Additionally, I would need a dropdown page, as I had seperated my portfolio to account for the different types of art I engage in. I would need to create a set up that would allow users to access the whole site through the navbar, and in order to do, I would have to create a dropdown for the links. This created a whole host of problems, which only mounted upon the issues that lay beyond. As Image e indicates, the spacing was the biggest issue with the over site, and even now, on smaller screens, it continues to bear down on the user. The site was designed and tested as the resolution of 1920x1080. Fortunately, as this current build of the site indicates, there was a way around it.

Anyways, those are my notes on the subject the redesign. Everything was done via a combo of CSS+HTML5, with only three lines of deprecated code written in Javascript as a result.
